Writing good skills

  • Use Rules sections for transformations the model must always apply.
  • Use Examples to show exact input → output pairs.
  • Keep skills focused on one domain — combine multiple skills on the agent instead.
  • Skills are passive knowledge only — they cannot trigger plugins or API calls.

📋 stdin/stdout JSON Protocol
Your script receives {"input":"...","parameters":{...},"userId":"..."} on stdin.
It must write {"success":true,"message":"...","data":...} to stdout.
Exit code 0 = success, non-zero = failure.
Available plugins
InputProcessor
Runs before the LLM — reads uploaded files (Excel, CSV, JSON), downloads URLs, and converts them to text the model can process. Enable when your agent accepts file or URL input.
OutputIntegrator
Runs after the LLM — validates output against a schema, then pushes results to an external API or database. Requires ApiEndpoint in request metadata. Enable only when the agent must write results to an external system.
Execution order: InputProcessor → LLM → OutputIntegrator. Plugins perform real I/O — skills cannot activate them.
